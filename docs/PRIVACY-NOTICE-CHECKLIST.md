# Privacy Notice Checklist (Third-Party/Public Prospect Data)

Use this when drafting/updating your privacy notice for the internal prospect research workflow.

## Include

- Identity of your business (controller)
- Contact details
- Purpose of processing (prospect research / direct marketing)
- Lawful basis (and brief explanation)
- Categories of personal data used
- Data sources (e.g. public LinkedIn pages, search engines, third-party data providers)
- Who data is shared with (e.g. Bright Data, hosting providers, analytics if any)
- Retention periods / review criteria
- Individual rights (including right to object to direct marketing)
- How to object / request deletion
- Whether profiling is used (and a plain-English description)
- International transfers (if providers/hosting process outside UK)

## MVP-Specific Notes

- State that the tool is used internally for targeted prospect research.
- State that activity/posts may be reviewed and stored as short-lived snapshots.
- State that suppression/do-not-contact records may be retained to honor objections.
- Avoid claiming “no storage” if CSV exports or saved prospects are used.

## Process Checklist

- [ ] Draft updated notice
- [ ] Review against actual tool behavior (saved prospects, activity snapshots, CSV export)
- [ ] Confirm objection email/process owner
- [ ] Publish notice URL
- [ ] Include notice link in first-contact messaging process
